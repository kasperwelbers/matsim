Now, using a large matrix (50,000 rows), that would normally crash
unless you have a good chunk of memory (worst case scenario: 50.000 \*
50.000 \* 8 bytes). Here we use the top\_n filter, for which the
required memory is at most nrow(m) \* top\_n \* 8 bytes. We also use the
verbose option to display a progress bar.

    m = abs(Matrix::rsparsematrix(50000,20000,0.001))

    cp = tcrossprod_sparse(m, top_n = 10, verbose=T)

Finally, we'll use the group/date filter, which very large matrices
(1,000,000 rows) to be processed by limiting comparisons to articles
within the same group and/or date range.

    m = abs(Matrix::rsparsematrix(1000000,20000,0.001))

    date = seq.Date(as.Date('1000-01-01'), as.Date('4010-01-10'), by=1)[1:nrow(m)] ## just to make 1,000,000 days
    cp = tcrossprod_sparse(m, date=date, lwindow = 15, rwindow=15, verbose=T)
